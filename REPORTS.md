# Отчёты

## Описание

В приложении реализована поддержка трёх типов отчётов через локальный прокси-сервис:

- `POST /report/csv` — формирует CSV-файл
- `POST /report/docx` — формирует DOCX-файл
- `POST /report` — формирует ODT-файл

Кнопки отчётов в UI отправляют запросы на локальный сервер по адресу `/report/...`, который проксирует их к внешнему сервису отчётов `http://185.192.247.60:8888`.

Прямой вызов из браузера на `http://185.192.247.60:8888` не работает из-за CORS-preflight, поэтому нужна прокси-схема.

## Пользовательский интерфейс

В разделе "Редактор данных" после выбора таблицы внизу появляются кнопки:

- `Добавить`
- `Редактировать`
- `Отчёт CSV`
- `Отчёт docx`
- `Добавить с копированием`
- `Удалить`
- `Показать карту`

Кнопки `Отчёт CSV` и `Отчёт docx` добавлены под блоком "Добавить / Редактировать" для аккуратного размещения.

При нажатии на кнопку отчет формируется по текущей таблице и скачивается в браузере.

## Формат тела запроса

### CSV

`POST /report/csv`

Content-Type: `application/json`

Пример:

```json
{
  "header": {
    "col1": "Название",
    "col2": "Значение"
  },
  "rows": [
    {"col1": "Строка 1", "col2": "100"},
    {"col1": "Строка 2", "col2": "200"}
  ]
}
```

### DOCX

`POST /report/docx`

Content-Type: `application/json`

Пример:

```json
{
  "filename": "my_report.docx",
  "elements": [
    {
      "type": "title",
      "text": "Заголовок отчёта",
      "formatting": {
        "font_name": "Arial",
        "font_size": 16,
        "alignment": "center",
        "bold": true
      }
    },
    {
      "type": "paragraph",
      "text": "Первый абзац текста.",
      "formatting": {
        "font_size": 12
      }
    },
    {
      "type": "table",
      "headers": ["Колонка 1", "Колонка 2"],
      "rows": [
        ["Ячейка 1", "Ячейка 2"],
        ["Ячейка 3", "Ячейка 4"]
      ]
    },
    {
      "type": "image",
      "bytes": "data:image/png;base64,iVBORw0KGgoAAAANS...",
      "caption": "Подпись к картинке"
    }
  ]
}
```

### ODT

`POST /report`

Content-Type: `application/json`

Пример:

```json
{
  "path_template": "template_name",
  "data": {
    "header": "Заголовок отчёта",
    "items": [
      {"name": "Пункт 1", "value": "100"},
      {"name": "Пункт 2", "value": "200"}
    ],
    "image_block": {
      "type": "image",
      "image_base64": "data:image/png;base64,iVBORw0KGgoAAAANS..."
    }
  }
}
```

## Использование в других местах

Для повторного использования можно вызвать локальный маршрут:

- `POST /report/csv`
- `POST /report/docx`
- `POST /report`

Если нужно подключить отчет из другого модуля, используйте клиентскую логику из `website/static/js/reportService.js`.
